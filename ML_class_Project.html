Introduction

 

                This report analyzes data from accelerometers to determine if dumbbell curls are performed correctly or incorrectly in one of 5 different ways. Accelerometers are placed on the arm, forearm, belt, and dumbbell. 6 participants are then asked to perform the curls either correctly or incorrectly and the data from the accelerometers is collected (1).

                                                                    Procedure

For the first part of the procedure the 49 columns containing the non-interval data and the classe set that will be predicted are loaded from the original training set. This data is then split into training and test sets. The test set will help in choosing predictors and cross validating the different models to estimate the out of sample error. Both training and test sets are preprocessed using the center and scale methods.

The next step is to select predictors. For this we need to balance the need for accuracy with the need for efficiency. The random forest method is used to rank the predictors using the Gini method. A group of top scoring predictors will be used to build a new model and the accuracy of that model will be compared to the model using all the predictors (Figure 1).

                The random forest model using all 48 predictors gives an accuracy of 99.04% for predicting the test set. Using the top 10 scoring predictors gives an accuracy of 93.78%. When the top 15 predictors are used, the accuracy goes up to 97.12%. The model with 15 predictors is about 2 percentage points less accurate than the original model but it only uses 1/3 the number of predictors (Figure 2).  It is determined that the data set using the 15 predictors will be used to build models in the caret package.

                For the next step, models are built with the caret package using the 15 predictors from the last step. Since the random forest method performed well in the previous phase, it is the first method used. Next the same data is fit using the gbm boosting method. Additional methods are also fit with the training data. K-Nearest-Neighbors (knn), Support Vector Machine (svmPoly), R-Partition (rpart), and Linear Discriminant Analysis(lda) are also used with their default settings.

                As expected, the Random Forest model in the caret package performed the beast at 97.12 % accuracy. The Knn and svmPoly also performed well at 94.25% and 92% respectively. Gbm method did ok at 89.35%. The worst preforming models were the lda and rpart models at 57.34% and 49.24% respectively (Figure 3). In the next step three different stacked models are created to determine if improvements can be made on the accuracy.

                After fitting the different caret models, three stacked models are created to see if the accuracy can be improved upon. The first stacked model combined the Random Forest and Generalized Boosting predictions. The Accuracy for this model was very poor at 47.22%. The next model combined the Random Forest and K Nearest Neighbor predictions. This model performed much better with an accuracy of 96.34 % which is above any single caret method but still below the Random Forest Model in R. Adding the Support Vector Machine to this model brought the accuracy up to 96.83% but it is still lower than the accuracy of the original Random Forest regression in R (Figure 4).

                Since the original Random Forest Method had the greatest accuracy with the default hyperparameters, an effort was made to optimize the mtry hyperparameter using the tuneRF function. The mtry hyperparameter represents the number parameters used at each split when creating the trees. An mtry value of 3 was found to be the best. When it was used the accuracy was the same at 97.12%.

 

                                                Conclusions

                In conclusion the final model is the Random Forest Regression in R with 15 parameters. It should be noted that the selection of the top 15 parameters for the model is somewhat arbitrary. It was inferred from the nature of the problem that speed would be an important factor as well as accuracy. In cases where speed is less of a factor, more parameters can be used to gain greater accuracy. The accuracy rate of 97.12% gives an estimated out of sample error rate of 2.88%. The actual out of sample error rate would most likely be a few percentage points higher due to overfitting.
